Overview & Architecture

The Maximo REST APIs are integrated starting from version 7.6.0.2 and use the same codebase as the OSLC APIs. They support end-to-end JSON with no namespace requirement (lean mode).

02

Authentication

POST

Maximo Native Auth

/oslc/login

Maximo manages the user repository. Credentials are passed as a Base64-encoded userid:password in the MAXAUTH header.

Request
HTTP
POST /oslc/login
maxauth: dXNlcjpwYXNz  # base64(user:pass)
POST

LDAP / BASIC Auth

/oslc/login

App-server manages authentication. Credentials passed via the standard HTTP Authorization header.

Request
HTTP
POST /oslc/login
Authorization: BASIC dXNlcjpwYXNz
POST

FORM Auth

/j_security_check

Form-encoded POST. Response includes jsessionid and LTPA token cookies that must be replayed in subsequent calls.

Request
HTTP
POST /j_security_check
Content-Type: application/x-www-form-urlencoded

j_username=wilson&j_password=wilson
GET

Logout

/oslc/logout

Terminates the server-side session. Not required when using API keys (stateless/silent login).

Request
HTTP
GET /oslc/logout
03

API Keys

API keys enable stateless/headless calls without user credentials. Each user can have one key; calls run in the context of that user and respect all permissions. No /logout needed.

POST

Create API Key (for logged-in user)

/oslc/apitoken/create
Request Body
JSON
{
  "expiration": -1   // -1 = never expires. Use minutes for expiry time
}
POST

Create API Key (admin — for any user)

/oslc/os/mxapiapikey
Request Body
JSON
{
  "expiration": -1,
  "userid": "WILSON"
}
POST

Revoke API Key

/oslc/apitoken/revoke

Revokes the API key for the currently authenticated user. No body required.

DELETE

Delete API Key (admin)

/oslc/os/mxapiapikey/{id}

Admin deletes a specific API key by its record ID.

GET

Using API Key in Requests

/oslc/os/mxapiasset?apikey=<value>&lean=1

Pass the key as a query param or as a request header. Header is preferred for security.

Option 1 — Query Param
HTTP
GET /oslc/os/mxapiasset?apikey=abc123&lean=1
Option 2 — Header (Recommended)
HTTP
GET /api/os/mxapiasset?lean=1
apikey: abc123
x-public-uri: http://host:port/maximo/api
Note: For app-server auth deployments, use the /api route instead of /oslc — it is not security-constrained, enabling apikey-based calls with SAML/OIDC setups.
04

Query (GET) — Selecting Data

GET

Get Collection (all records)

/oslc/os/mxapiasset?oslc.pageSize=10&lean=1

Returns a paged collection of asset records. Each member contains only an href link by default unless oslc.select is used.

Response Structure
JSON Response
{
  "member": [
    { "href": "http://host/maximo/oslc/os/mxapiasset/_QTYwMDI..." },
    { "href": "http://host/maximo/oslc/os/mxapiasset/_TDEyNTEw..." }
  ],
  "responseInfo": {
    "href": "...",
    "nextPage": { "href": "..." },
    "pagenum": 1,
    "totalCount": 250   // only if collectioncount=1
  }
}
GET

Select Specific Attributes

/oslc/os/mxapiasset?oslc.select=assetnum,location,description,status&lean=1

Use oslc.select to specify which attributes to return. Domain-bound attributes automatically include a _description companion field (e.g. status_description).

Example with page size
GET /oslc/os/mxapiasset?oslc.pageSize=2
  &oslc.select=assetnum,location,description,status
  &lean=1
GET

Select Child Object Attributes

/oslc/os/mxapiasset?oslc.select=assetnum,assetmeter{*}

Use curly-brace notation to inline child object data. * selects all attributes; or list them explicitly. Supports multiple nesting levels.

All child attributes
Select Clause
oslc.select=assetnum,status,assetmeter{*}
Nested child objects
Nested
oslc.select=obj1{prop1,obj2{attr1,attr2}}
GET

Select Related Object Attributes (Dot Notation)

/oslc/os/mxapiasset?oslc.select=assetnum,location.description,location.status

Dot notation fetches attributes from related MBOs not in the object structure. Use aliasing (--) when attribute name clashes with relation name.

Dot notation (1:1)
Example
oslc.select=assetnum,
  location--mylocation,
  location.status,
  location.description
rel notation (1:many)
Example
oslc.select=assetnum,
  rel.openwo{wonum,description}
&oslc.where=openwo.wonum="*"
GET

Traverse to Related MboSet

/oslc/os/mxapiasset/{rest id}/openwo?oslc.select=*

Use a relationship name as a URI token to navigate to a related collection. Can be chained recursively.

Examples
HTTP
# Navigate to work orders of an asset
GET /oslc/os/mxapiasset/{rest id}/openwo?oslc.select=*

# Return as a specific object structure
GET /oslc/os/mxapiasset/{rest id}/openwo.mxapiwodetail?oslc.select=*

# Chain deeper
GET /oslc/os/mxapiasset/{rest id}/openwo/{rest id}/jobplan?oslc.select=*
GET

Get Total Count Only

/oslc/os/mxapiasset?count=1
Count only
Response
{ "totalCount": 10 }
Count with results
Query Param
?collectioncount=1
GET

Formula Properties (Dynamic Attributes)

/oslc/os/mxapiasset?oslc.select=assetnum,exp.myreplacecost

Select object formula values using exp. prefix. No db config or admin mode required — great alternative to non-persistent attributes.

Response
{
  "assetnum": "L12510",
  "myreplacecost": 1450.0
}

Filtering — oslc.where

GET

Basic Where Filter

/oslc/os/mxapiasset?oslc.where=status="OPERATING"&lean=1

<Less thanlinecost<200.5

value%Starts withstatus="APPR%"
%valueEnds withstatus="%APPR"

!="*"IS NULLstatus!="*"
Examples
Where Clause Examples
# Multiple conditions
?oslc.where=status="OPERATING" and location.status="OPERATING"

# Date range + numeric + boolean
?oslc.where=priority>1 and priority<=3
  and installdate>="1999-02-06T00:00:00-05:00"
  and linear=false

# In clause with strings
?oslc.where=status in ["OPERATING","ACTIVE"]

# In clause with numbers
?oslc.where=priority in [1,2,3]
GET

Timeline Filter

/oslc/os/mxapiwodetail?tlrange=-3M&tlattribute=reportdate

Simplified date-range queries indexed on current date. Supports Y (years), M (months), D (days), and +/- directions.

Timeline Examples
# Work orders reported in the past 3 months
?tlrange=-3M&tlattribute=reportdate

# 3 years before AND after current date (range)
?tlrange=+-3Y&tlattribute=reportdate

# Index on a specific date instead of today
?tlattribute=reportdate=2024-01-01T00:00:00-05:00
GET

Classification Attribute Search

/oslc/os/mxapiasset?attributesearch=[SPEED:>=50]

Search assets/locations/items by their classification spec attributes.

Examples
# Assets with SPEED >= 50
?attributesearch=[SPEED:>=50]

# Assets that have SPEED attribute at all
?attributesearch=[SPEED]

# Multiple attributes (ANDed)
?attributesearch=[SPEED:>=50;AREA;ELEV:=300]
GET

SynonymDomain Internal Value Filter

/oslc/os/mxapiwodetail?domaininternalwhere=status=APPR,INPRG

Filter by internal domain values (base Maximo values, not customer-translated external values). Independent of oslc.where, ANDed to it if both present.

Examples
# WOs with status = APPR or INPRG (internal values)
?domaininternalwhere=status=APPR,INPRG

# WOs NOT in APPR or INPRG
?domaininternalwhere=status!=APPR,INPRG
06

Sorting & Paging

GET

Sort Results

/oslc/os/mxapiasset?oslc.orderBy=-attr1,%2Battr2

Prefix - for descending, + for ascending. URL-encode + as %2B.

Example
?oslc.orderBy=-assetnum,%2Bstatus
GET

Paging

/oslc/os/mxapiasset?oslc.pageSize=10

Response includes nextPage, previousPage, pagenum, optional totalCount/totalPages.

Stable Paging
?oslc.pageSize=10&stablepaging=1
Stable Paging: Retains MboSet in memory across pages (no repeated SQL). Pages expire after 5 minutes of inactivity or when set is fully paged. No backward navigation.

Create & Update Resources

POST

Create a Resource

/oslc/os/mxapiasset

POST to the collection URI. Response Location header contains the URI of the new record. Use properties header to return fields immediately.

Basic Create
HTTP
POST /oslc/os/mxapiasset
Content-Type: application/json

{
  "assetnum": "ASSET1",
  "siteid": "BEDFORD",
  "description": "my first asset"
}
Create + Return Fields
HTTP
POST /oslc/os/mxapiasset
properties: assetnum,status

{
  "assetnum": "ASSET1",
  "siteid": "BEDFORD",
  "description": "my first asset"
}
PATCH

Update a Resource

/oslc/os/mxapiasset/{rest id}

Use x-method-override: PATCH on a POST. Only the supplied fields are updated.

HTTP
POST /oslc/os/mxapiasset/{rest id}
x-method-override: PATCH
properties: *

{
  "location": "BR300",
  "description": "updated asset"
}
PATCH

Add Child Objects (MERGE)

/oslc/os/mxapiasset/{rest id} — patchtype: MERGE

MERGE: Matches existing children and updates them; creates new ones. Does NOT delete unmentioned children. PATCH (default): Deletes all children not in the payload.

HTTP — MERGE
POST /oslc/os/mxapiasset/{rest id}
x-method-override: PATCH
patchtype: MERGE
properties: *

{
  "assetmeter": [
    { "metername": "TEMP-F", "linearassetid": 0 },
    { "metername": "ABC", "linearassetid": 0 }
  ]
}
PATCH

Update Child Objects (set meter reading)

/oslc/os/mxapiasset/{rest id}

Include the child's primary key in the payload to identify which child to update. Or use the child href URI.

HTTP
POST /oslc/os/mxapiasset/{rest id}
x-method-override: PATCH
patchtype: MERGE

{
  "assetmeter": [
    {
      "metername": "TEMP-F",
      "linearassetid": 0,
      "newreading": "10"
    }
  ]
}
PATCH

Delete Child Object

/oslc/os/mxapiasset/{rest id} — _action: Delete
Via _action in payload
HTTP
{
  "assetmeter": [
    {
      "metername": "TEMP-F",
      "linearassetid": 0,
      "_action": "Delete"
    }
  ]
}
Via HTTP DELETE on child localref
HTTP
DELETE <assetmeter localuri>

Delete Resources

DELETE

Delete Resource (HTTP DELETE)

DELETE /oslc/os/mxapiasset/{rest id}

/oslc/os/{os}/{rest id}

{ "_action": "Delete" }
GET

Check if Resource Can Be Deleted

/oslc/os/{os}/{rest id}?action=system:candelete

Returns 200 OK if the MBO can be deleted. Requires the application MBO to implement canDelete().

09

Bulk & Sync

Bulk: Send multiple create/update/delete operations in a single HTTP request against the collection URI with _action per record. Sync: Upsert semantics — creates if not found, updates if exists. Both use POST to the collection URI.
POST

Bulk Create/Update

/oslc/os/mxapiasset
HTTP
POST /oslc/os/mxapiasset
x-method-override: BULK

[
  { "assetnum": "A001", "siteid": "BEDFORD", "_action": "Create" },
  { "assetnum": "A002", "siteid": "BEDFORD", "_action": "Create" }
]
POST

Sync (Upsert)

/oslc/os/mxapiasset
HTTP
POST /oslc/os/mxapiasset
x-method-override: SYNC

{ "assetnum": "A001", "siteid": "BEDFORD", "description": "updated" }

Actions

GET

List Allowed Actions on a Resource

/oslc/os/mxapiwo?oslc.where=wonum="1001"&oslc.select=allowedactions

Returns all available WebMethod and script actions registered to the object structure.

HTTP

Invokes a service method (annotated with @WebMethod) on the resource. The MBO is derived from the URI.

HTTP

Invoke an automation script registered as an action on the object structure. Script name used as action name.

HTTP

11

Automation Scripts (Custom APIs)

GET

Call a Custom Script API

/oslc/script/countofwoandsr?site=ABC

Automation scripts can expose custom REST endpoints. Any HTTP method can be used. The script handles input via request.getQueryParam() and sets the JSON response object.

Script (JavaScript)
Automation Script
importPackage(Packages.psdi.server);
var resp = {};
var site = request.getQueryParam("site");
var woset = MXServer.getMXServer()
  .getMboSet("workorder", request.getUserInfo());
woset.setQbe("siteid", "="+site);
resp.wocount = woset.count();
// ... set srcount similarly
resp.total = resp.wocount + resp.srcount;
Response
JSON
{
  "wocount": 100,
  "srcount": 20,
  "total": 120
}

Fetch Workflow Assignments

/oslc/os/mxapiwfassignment?oslc.select=*

Returns all workflow assignments for the authenticated user, including positive/negative action options.

Quick Reference Cheatsheet

All Key Endpoints at a Glance
## AUTH
POST   /oslc/login                                  # Login (MAXAUTH / BASIC / FORM)
POST   /oslc/apitoken/create                        # Create API key
POST   /oslc/apitoken/revoke                        # Revoke API key
GET    /oslc/logout                                 # Logout

## QUERY
GET    /oslc/os/{os}                                # Get collection
GET    /oslc/os/{os}/{id}                           # Get single record
GET    /oslc/os/{os}/{id}/{relation}                # Traverse relationship
GET    /oslc/os/{os}?oslc.select=f1,f2             # Select fields
